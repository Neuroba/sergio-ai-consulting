# Context Scan — быстрый контекст сессии

## Когда использовать

- Старт каждой сессии (см. CLAUDE.md)
- Sergio говорит: «контекст», «что сегодня», «скан дня», «что в работе»
- Нужно понять приоритеты перед началом работы

## Инструкции

### Шаг 1: Параллельные запросы (все одновременно)

Запусти ВСЕ запросы параллельно, не последовательно:

1. **Todoist** — `find-tasks-by-date` с `startDate: "today"`, `limit: 30`
2. **Notion Dashboard** — `notion-fetch` страницу `38a6a70c-e737-8139-9d56-e3bcb4e9ce8b`
3. **Notion поиск** — `notion-search` по запросу `В работе`, `page_size: 10`, `max_highlight_length: 80`

НЕ используй `notion-query-data-sources` — требует Business план.

### Шаг 2: Форматирование

Выведи результат в ОДНОМ сообщении, формат:

```
## Скан дня — [дата], [день недели]

### Todoist (сегодня + просрочка)
| | Задача | Приоритет | Проект |
|---|--------|-----------|--------|
| emoji | текст | p1-p4 | проект |

### Проекты (Notion Dashboard)
| Проект | Статус | Фокус |
|--------|--------|-------|
| emoji+название | цветной статус | фокус Q |

### В работе (Notion)
- [Проект] Задача
- [Проект] Задача

---
**Главный блокер:** [что мешает больше всего]
**Рекомендация:** [1 предложение — с чего начать день]
```

### Шаг 3: ID проектов для фильтрации Notion

При парсинге результатов поиска Notion фильтруй только страницы с `38a6a70c-e737-*` (рабочее пространство Claude), игнорируй старые крипто-страницы.

## Правила

- Максимально параллельные запросы — скорость важнее
- Не задавай вопросов, просто покажи контекст
- Если Todoist пуст — скажи прямо, не придумывай задачи
- Просроченные задачи выделяй отдельно
- Рекомендацию давай на основе блокеров и приоритетов
