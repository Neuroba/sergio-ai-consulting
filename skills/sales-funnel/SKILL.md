# Sales Funnel — аудит и создание конверсионных воронок

## Когда использовать

- Создание лендинга/сайта с целью конверсии (заявка, покупка, контакт)
- Аудит существующей страницы на конверсию: "проверь сайт", "почему не конвертит"
- Перестройка структуры сайта под воронку продаж
- Выбор фреймворка: PAS, AIDA, StoryBrand под конкретную ЦА

## Инструкции

### Шаг 1: Определи контекст

Собери перед аудитом/созданием:
- **ЦА:** кто конкретно приходит на страницу (сегмент, боль, бюджет)
- **Цель:** одно целевое действие (заявка? звонок? покупка?)
- **Канал:** откуда трафик (LinkedIn? Telegram? Google? реферал?)
- **Продукт:** что продаём, средний чек, порог входа

### Шаг 2: Запусти аудит (для существующих страниц)

```bash
python3 ~/.claude/skills/sales-funnel/tools/audit.py --url "path/to/file.html"
```

Скрипт проверяет 30 критериев конверсии и выдаёт оценку + список фиксов.

### Шаг 3: Выбери фреймворк

| ЦА знает проблему? | ЦА знает решение? | Фреймворк |
|:---:|:---:|---|
| ❌ Нет | ❌ Нет | **AIDA** (Attention → Interest → Desire → Action) |
| ✅ Да | ❌ Нет | **PAS** (Problem → Agitation → Solution) |
| ✅ Да | ✅ Да | **StoryBrand** (Guide → Plan → Success → Failure) |

### Шаг 4: Построй/перестрой секции

Порядок секций (универсальный, адаптировать под фреймворк):

```
1. HERO         — Боль клиента, не фича продукта. 1 CTA.
2. PAIN         — 3-4 конкретные боли ЦА с цифрами
3. SOLUTION     — Как именно решаем (визуально, просто)
4. PROOF        — Кейсы Before/After, цифры, testimonials
5. HOW          — 3 шага процесса (снимает "сложно")
6. PRICING      — От бесплатного/дешёвого к дорогому
7. FAQ          — 5-6 возражений → ответы
8. CTA          — Повтор главного действия, zero-risk
```

### Шаг 5: Проверь по чек-листу конверсии

Запусти `tools/audit.py` на результат и убедись что score ≥ 85/100.

## Принципы

- Один сайт = одно целевое действие (не "позвони/напиши/подпишись")
- Hero без фич — только боль и обещание результата
- Каждая секция снимает одно возражение
- Social proof ДО pricing (сначала доверие, потом цена)
- CTA повторяется минимум 3 раза по странице
- Mobile-first (60%+ трафика с телефона)
