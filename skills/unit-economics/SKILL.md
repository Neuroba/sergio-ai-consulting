# Unit Economics — калькулятор юнит-экономики

## Когда использовать

- Пользователь спрашивает про маржу, наценку, breakeven
- Нужно рассчитать план продаж для целевой прибыли
- Строится система мотивации продавца (база + комиссия)
- Сравниваются сценарии дня/недели/месяца
- Анализ «сколько нужно продать, чтобы заработать X»

## Инструкции

### Шаг 1: Собери входные данные

Обязательные:
- **products**: список продуктов — название, цена продажи, себестоимость
- **fixed_costs**: постоянные затраты за период (аренда, база продавца, и т.д.)
- **target_profit**: целевая чистая прибыль за период

Опциональные:
- **seller_base**: фиксированная оплата продавца за смену (по умолчанию 0)
- **commission_rate**: процент комиссии сверх плана (по умолчанию 0.25)
- **product_mix**: предполагаемая доля каждого продукта в выручке

### Шаг 2: Запусти скрипт

```bash
python3 ~/.claude/skills/unit-economics/tools/calc.py \
  --products '[{"name":"50g","price":8,"cost":2.5},{"name":"250g","price":35,"cost":12.5}]' \
  --fixed-costs 76 \
  --target-profit 300 \
  --seller-base 20 \
  --commission-rate 0.25
```

Скрипт выдаёт:
1. Таблицу маржи по каждому продукту
2. Breakeven и план-порог выручки
3. 4 сценария дня (слабый / план / хороший / сильный)
4. Расчёт оплаты продавца в каждом сценарии
5. Фразу для продавца «объясни за 10 секунд»

### Шаг 3: Интерпретируй результат

- Если план нереалистичен (слишком много единиц) — скажи об этом прямо
- Предложи рычаги: апсейл на дорогой продукт, комплекты, дегустации
- Если пользователь хочет Excel — предложи скилл `excel-model`

## Принципы

- **Честность:** если целевая прибыль недостижима при данном трафике — скажи прямо
- **Простота:** система мотивации должна помещаться на один листок
- **Защита владельца:** комиссия только с прибыли СВЕРХ целевой
- **Валюта:** всегда уточняй (USD / ARS / другая)
